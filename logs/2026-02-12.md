## 다음 세션에서 반드시 복습할 것

### 틀린 것 (오답)

1. **`-`로 시작하는 파일 삭제법** (Q10)
   - ❌ 따옴표나 이스케이프로는 안 됨! (`rm "-file.txt"` 여전히 옵션으로 해석)
   - ✅ `rm -- -file.txt` (-- 뒤는 전부 파일명)
   - ✅ `rm ./-file.txt` (경로를 붙여서 해결)
   - 핵심: 따옴표는 셸 확장을 막는 거고, `-` 옵션 해석은 명령어 자체의 문제

2. **`$()` vs `$(())` 혼동** (Q17)
   - `$()` = 명령어 치환 (명령어 실행 → 출력 결과로 대체)
   - `$(())` = 산술 확장 (수학 계산 → 결과로 대체)
   - ❌ `$()`를 변수 확장이라고 답함

### 부분 정답 (애매했던 것)

3. **`rm -rf`가 위험한 이유** (Q8)
   - `-r`: 재귀적으로 모든 하위 파일/디렉토리 삭제
   - `-f`: 확인 없이 강제 삭제 ← 이걸 빠뜨림
   - 리눅스에는 휴지통이 없어서 복구 불가

4. **`2>&1` 순서 문제** (Q13)
   - `> file 2>&1` → stdout을 file로 → stderr도 file로 → 둘 다 file
   - `2>&1 > file` → stderr를 화면으로 → stdout만 file → stderr는 화면에 남음
   - 핵심: `2>&1`은 실행 시점의 stdout 위치를 따라감

5. **산술 확장은 정수만** (Q16)
   - `$((15 % 4))` = 3 (나머지 연산)
   - ❌ 3.75라고 답함 (소수점 나눗셈으로 착각)
   - `$(())`는 정수 연산만 지원!

### 몰랐던 것 (미답변)

6. **bash 풀네임**: Bourne-Again Shell (원조: sh, Bourne Shell)
7. **프롬프트 `$` vs `#`**: `$` = 일반 사용자, `#` = root
8. **`ls -l` 맨 앞 글자**: `d` = 디렉토리, `-` = 일반 파일, `l` = 심볼릭 링크
9. **less 조작**: `G` = 맨 끝, `g` = 맨 처음, `/단어` = 검색, `q` = 나가기
10. **`cp -r` 없으면**: 디렉토리 복사 불가 (에러 발생)
11. **하드 링크 vs 심볼릭 링크 차이 3가지**:
    - 원본 삭제 시: 하드=유지, 심볼릭=깨짐
    - 디렉토리에: 하드=불가, 심볼릭=가능
    - 다른 디스크: 하드=불가, 심볼릭=가능
12. **cd가 내장인 이유**: 외부 프로그램이면 자식 프로세스에서 디렉토리 변경 후 사라짐, 원래 셸은 그대로
13. **큰따옴표 안에서 되는/안되는 확장**:
    - 되는 것: `$변수`, `$(명령)`, `$((식))` → $ 계열만!
    - 안되는 것: `*`, `~`, `{a,b}`
