## Ch.9 Permissions 학습 내용

### 핵심 개념
- 권한 구조: owner / group / others × r / w / x
- 권한 체크 순서: owner → group → others (첫 매칭에서 끝)
- group = 사용자들의 팀. 파일에는 그룹 1개만 지정

### chmod
- 숫자 방식: r=4, w=2, x=1 더해서 3자리 (755, 644, 600)
- 기호 방식: `u+x`, `g-w`, `o-rwx`
- 숫자 = 통째로 설정, 기호 = 하나 추가/제거

### sudo / su / chown
- sudo: 명령어 하나만 관리자로 (실무에서 주로 사용)
- su: 사용자 전환 (거의 안 씀)
- chown: 소유자/그룹 변경 (sudo 필요)

### 퀴즈 결과: 5문제 중 4개 정답
- 틀린 것: chown에 sudo 필요한 이유 — "남의 파일 소유권을 가로챌 수 있어서"가 핵심

## Ch.10 Processes 학습 내용

### 핵심 명령어
- `ps aux`: 전체 프로세스 목록
- `top`: 실시간 모니터링
- `&`: 백그라운드 실행
- `Ctrl+Z` → `bg`: 일시 정지 → 백그라운드 재개
- `fg %번호`: 포그라운드로 가져오기
- `kill PID` / `kill -9 PID`: 종료 요청 / 강제 종료

### 실습 중 배운 것
- `fg/bg`는 jobs 번호(`%1`) 사용, `kill`은 PID 사용 — 처음에 `fg 23222`로 해서 에러남
- `ps aux | grep sleep` 하면 grep 자기 자신도 잡힘 (정상)

### 퀴즈 결과: 5/5 만점!
