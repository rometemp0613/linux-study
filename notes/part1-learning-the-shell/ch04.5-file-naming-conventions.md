# Ch.4.5: 파일명 작명법

## 리눅스 파일명 기본 규칙

- 대소문자 구분: `File.txt` ≠ `file.txt`
- 숨김 파일: `.`으로 시작 (`.bashrc`)
- 확장자에 특별한 의미 없음 (관례일 뿐)
- 최대 길이: 255자

## 좋은 파일명 규칙

| 규칙 | 좋은 예 | 나쁜 예 |
|------|--------|--------|
| 공백 대신 `-` 또는 `_` | `my-report.txt` | `my report.txt` |
| 소문자 사용 | `my-report.txt` | `MyReport.txt` |
| 안전한 문자만 | `report-v2.txt` | `report&(v2).txt` |
| 날짜는 YYYY-MM-DD | `report-2026-01-30.txt` | `report-1-30-26.txt` |
| `-`로 시작 금지 | `backup.tar` | `-backup.tar` |

## 나쁜 파일명이 위험한 이유

```bash
# 공백 → 여러 인자로 해석
rm my report.txt    # "my"와 "report.txt" 두 파일 삭제 시도

# -로 시작 → 옵션으로 해석
rm -file.txt        # -f, -i, -l, -e 옵션으로 해석됨

# 특수문자 → 셸 문법과 충돌
touch hello;rm -rf ~   # 명령어 주입 위험
```

## 공백 있는 파일 다루기

```bash
# 방법 1: 따옴표
cp "my file.txt" backup/

# 방법 2: 백슬래시 이스케이프
cp my\ file.txt backup/

# Tab 자동완성 활용하면 자동 이스케이프!
```

## -로 시작하는 파일 다루기

```bash
rm -- -file.txt     # -- 뒤는 전부 파일명으로 해석
rm ./-file.txt      # 경로를 붙여서 해결
```
