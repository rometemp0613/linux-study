# Ch.10: Processes (프로세스)

## 프로세스란?

```
프로그램 = 파일 (디스크에 저장된 코드)
프로세스 = 실행 중인 프로그램 (메모리에 올라간 상태)
```

- 각 프로세스에는 **PID(Process ID)**라는 고유 번호가 붙음

## 프로세스 확인

```bash
ps            # 현재 터미널의 프로세스만
ps aux        # 시스템 전체 프로세스 (가장 많이 씀)
top           # 실시간 모니터링 (q로 나가기)
```

### ps aux 주요 컬럼

- USER: 실행한 사용자
- PID: 프로세스 ID
- %CPU / %MEM: CPU/메모리 사용률
- COMMAND: 실행 중인 명령어

## 백그라운드/포그라운드

```bash
sleep 100 &     # 처음부터 백그라운드에서 실행
jobs            # 백그라운드 작업 목록
fg %1           # 1번 작업을 포그라운드로
bg %1           # 정지된 작업을 백그라운드에서 재개
Ctrl+Z          # 실행 중인 프로그램 일시 정지
Ctrl+C          # 실행 중인 프로그램 종료
```

### 흐름 정리

```
명령어 &     → 백그라운드 실행
Ctrl+Z       → 일시 정지
bg %번호     → 정지된 거 백그라운드에서 재개
fg %번호     → 포그라운드로 가져오기
```

### 주의: fg/bg vs kill

- **fg, bg** → jobs 번호 사용 (`fg %1`)
- **kill** → PID 사용 (`kill 12345`)

## 프로세스 종료

```bash
kill 12345        # 종료 요청 (정상 종료, 정리 시간 줌)
kill -9 12345     # 강제 종료 (즉시 죽임, 최후의 수단)
killall firefox   # 이름으로 전부 종료
```

## 자주 쓰는 패턴

```bash
ps aux | grep 이름     # 특정 프로세스 찾기
top                     # 뭐가 CPU 잡아먹는지 확인
kill -9 PID             # 응답 없는 프로세스 강제 종료
```
