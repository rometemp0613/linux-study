# Chapter 3: Exploring the System

## ls 명령어 심화

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-l` | 긴 형식 (자세한 정보) |
| `-a` | 숨김 파일 포함 (`.`으로 시작하는 파일) |
| `-h` | 사람이 읽기 쉬운 크기 (KB, MB) |
| `-t` | 수정 시간순 정렬 |
| `-r` | 역순 정렬 |
| `-S` | 파일 크기순 정렬 |

옵션 조합 가능: `ls -lah`

### ls -l 출력 해석

```
drwxr-xr-x  5  user  group  4096  Jan 30 10:00  Documents
│           │  │     │      │     │             │
│           │  │     │      │     │             └─ 이름
│           │  │     │      │     └─ 수정 날짜/시간
│           │  │     │      └─ 파일 크기 (바이트)
│           │  │     └─ 그룹
│           │  └─ 소유자
│           └─ 링크 수
└─ 파일 유형 + 권한
```

**첫 글자 의미:**
- `d` = 디렉토리
- `-` = 일반 파일
- `l` = 심볼릭 링크

## file 명령어

### 왜 필요한가?
- Linux는 확장자로 파일 유형을 결정하지 않음
- 파일의 **실제 내용**을 분석해서 유형 판단
- 확장자가 없거나 거짓일 수 있음

### 사용법

```bash
file 파일경로
```

### 예시

```bash
file document.txt      # Unicode text, UTF-8 text
file /bin/ls           # Mach-O 64-bit executable (실행 파일)
file video.mp4         # ISO Media, MP4 Base Media
file /home             # directory
```

### 언제 쓰나?
- 확장자 없는 파일이 뭔지 모를 때
- 다운받은 파일이 진짜 그 형식인지 확인할 때
- 텍스트 파일인지 바이너리인지 구분할 때

## less 명령어

### 왜 필요한가?
- `cat`은 파일 전체를 한 번에 출력
- 파일이 크면 화면이 지나가버려서 보기 어려움
- `less`는 **페이지 단위**로 위아래 이동하며 볼 수 있음

### 사용법

```bash
less 파일경로
```

### less 조작법

| 키 | 동작 |
|-----|------|
| `Space` / `f` | 다음 페이지 |
| `b` | 이전 페이지 |
| `↑` / `↓` 또는 `j` / `k` | 한 줄씩 이동 |
| `g` | 파일 맨 처음으로 |
| `G` | 파일 맨 끝으로 |
| `/단어` | 단어 검색 |
| `n` | 다음 검색 결과 |
| **`q`** | **종료 (필수!)** |

### 핵심
- `less`에 들어가면 별도의 화면이 열림
- **`q`를 눌러야 나올 수 있음**
- "Less is more" - `more` 명령어의 개선판

## 심볼릭 링크 (Symbolic Link)

### 개념
- **Windows의 "바로가기"**와 동일
- 실제 파일/폴더가 아니라 다른 곳을 가리키는 포인터
- 원본 경로로 자동 연결됨

### 왜 필요한가?
- 긴 경로를 짧게 단축
- 버전 관리 (예: `python -> python3.9`)
- 호환성 유지

### 확인 방법

```bash
ls -l /tmp
# 출력: lrwxr-xr-x  ...  /tmp -> private/tmp
#       │                 │       │
#       │                 │       └─ 실제 위치
#       │                 └─ 심볼릭 링크
#       └─ 'l' = 심볼릭 링크 표시
```

### 만드는 법

```bash
ln -s 원본경로 링크이름
```

예시:
```bash
ln -s /home/me/documents/important-file.txt ~/quick-access
```

## 주요 시스템 디렉토리

| 디렉토리 | 설명 |
|----------|------|
| `/` | 루트, 모든 것의 시작 |
| `/bin` | 필수 바이너리 (ls, cp 등) |
| `/boot` | 부팅에 필요한 파일 |
| `/dev` | 장치 파일 (디스크, 키보드 등) |
| `/etc` | 시스템 설정 파일 |
| `/home` | 사용자 홈 디렉토리들 |
| `/lib` | 공유 라이브러리 |
| `/root` | root 사용자의 홈 |
| `/tmp` | 임시 파일 |
| `/usr` | 사용자 프로그램, 문서 |
| `/var` | 로그, 메일 등 가변 데이터 |
