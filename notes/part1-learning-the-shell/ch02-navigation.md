# Chapter 2: Navigation

## 핵심 개념

### 파일 시스템 트리
- Linux는 **단일 파일 시스템 트리** 구조
- 모든 것이 **루트(`/`)**에서 시작
- Windows처럼 `C:\`, `D:\` 드라이브 개념이 없음

```
/
├── bin      # 필수 바이너리
├── etc      # 설정 파일
├── home     # 사용자 홈 디렉토리들
│   └── me
├── usr      # 사용자 프로그램
└── ...
```

---

## 핵심 명령어

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `pwd` | 현재 작업 디렉토리 출력 | `/home/me` |
| `ls` | 디렉토리 내용 나열 | `ls /usr` |
| `cd` | 디렉토리 변경 | `cd /home` |

---

## 경로 (Path)

### 절대 경로 (Absolute Path)
- 루트(`/`)에서 시작하는 전체 경로
- 어디서든 동일하게 작동
- 예: `/home/me/Documents/file.txt`

### 상대 경로 (Relative Path)
- 현재 위치 기준으로 표현
- 현재 위치에 따라 의미가 달라짐
- 예: `Documents/file.txt`

---

## 특수 디렉토리 표기

| 표기 | 의미 | 사용 예 |
|------|------|---------|
| `.` | 현재 디렉토리 | `./script.sh` |
| `..` | 상위(부모) 디렉토리 | `cd ..` |
| `~` | 홈 디렉토리 | `cd ~/Documents` |
| `-` | 이전 작업 디렉토리 | `cd -` |

---

## cd 명령어 활용

```bash
cd                  # 홈 디렉토리로 이동
cd ~                # 홈 디렉토리로 이동 (위와 동일)
cd /usr/bin         # 절대 경로로 이동
cd Documents        # 상대 경로로 이동
cd ..               # 상위 디렉토리로 이동
cd ../..            # 두 단계 상위로 이동
cd -                # 이전 디렉토리로 돌아가기
```

---

## 파일/디렉토리 이름 규칙

1. **대소문자 구분** - `File.txt` ≠ `file.txt`
2. **숨김 파일** - `.`으로 시작하면 숨김 파일 (예: `.bashrc`)
3. **공백 허용** - 가능하지만 따옴표 필요: `"my file.txt"`
4. **확장자** - Linux는 확장자로 파일 유형 결정 안 함 (내용으로 판단)
